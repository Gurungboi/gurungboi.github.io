Architecture

I've driven iOS through three full architectural generations at Prezzee — not as academic exercises, but as direct responses to real scaling pain. Each migration was proposed, designed, documented, and rolled out under my ownership. I wrote the migration guides, ran workshop sessions, and paired through the first features to make each pattern stick across the team.

"Architecture should solve the problems you actually have — not the ones that look good on a whiteboard."

— My approach at every migration decision
Generation 01
MVC
2019 — 2020
The inherited codebase. ViewControllers owning everything — networking, state, navigation, UI. Worked fine at small scale but testing was painful and onboarding slow.
Massive ViewControllers (1000+ lines)
Tight coupling between UI and business logic
Unit testing nearly impossible
Why move on?
Codebase growing, team growing — couldn't scale
→
Generation 02
MVVM + Coordinator
2021 — 2023
Separated concerns properly. ViewModels handled logic, Coordinators owned navigation. Testability improved dramatically and new engineers ramped up faster.
Combine-powered reactive bindings
Coordinator pattern for decoupled navigation
ViewModels fully unit-testable
Why move on?
State complexity caused subtle, hard-to-reproduce bugs
→
Generation 03 · Current
MVI + Coordinator
2024 — Present
Unidirectional data flow. Intent → State → Reducer loop makes side effects predictable, screen state trivial to test, and onboarding new engineers even faster.
Single source of truth for screen state
Predictable side-effect handling
Easier debugging — state is always reproducible
Result
Fewer state-related bugs, faster feature delivery
